# Planner Acceptance And Documentation Impact

Load this skill only when defining completion conditions for the plan.

## Required Actions

- Write observable acceptance criteria for each subtask.
- Keep acceptance criteria focused on implementation outcomes, not downstream workflow behavior.
- Add an overall `Documentation Impact` section for the feature.
- Add a per-subtask `Documentation Impact` section or explicitly state when no documentation update is expected.

## Limits

- Do not encode Tester, Documenter, Verifier, or Reviewer workflow actions as implementer acceptance criteria.
