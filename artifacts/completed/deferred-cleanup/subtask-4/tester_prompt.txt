## What Was Implemented

`BlogService.create()` now catches MySQL ER_DUP_ENTRY (errno 1062) and SQLite
"UNIQUE constraint failed" errors on the blog post save when using auto-derived
slugs, and retries `deriveUniqueSlug` + `save()` up to 3 times (SLUG_RETRY_LIMIT).
On exhaustion a `ConflictException` (HTTP 409) is thrown. Non-duplicate-key errors
propagate unchanged. The explicit-slug path in `create()` is not affected.

New private helpers added:
- `BlogService.isDuplicateKeyError(err)` — classifies dup-key errors
- `BlogService.saveWithDerivedSlugRetry(...)` — retry loop

deriveUniqueSlug call sites covered: 1 (create() only; update() never calls
deriveUniqueSlug).

## Acceptance Criteria

1. Save rejected with duplicate-key error retries derivation and succeeds with
   suffixed slug — driven by executed unit test mocking rejection-then-success.
2. Retry attempts are bounded; exhaustion returns controlled error envelope (409),
   never raw 500.
3. Every deriveUniqueSlug-consuming write path is covered; count stated (1).
4. JSDoc on affected methods reflects retry semantics.
