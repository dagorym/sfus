## Implementation Context

- `PublicTopicShape` in `forums.types.ts` now has `lastPostAuthor: PublicAuthorShape | null`.
- `ForumsService.toTopicShape` now accepts a second optional `lastPostAuthor` parameter (defaults to `null`), so `createTopic` still works correctly.
- `ForumsService.resolveTopicLastActivityAuthors(topicIds, openingAuthors)` resolves last-reply authors for a batch of topic IDs in one grouped SQL query using a correlated `MAX(created_at)` subquery; soft-deleted posts are filtered out. Returns `Map<topicId, PublicAuthorShape | null>`.
- `ForumsService.listTopics` now calls `resolveTopicLastActivityAuthors` after fetching the page, and passes the result into `toTopicShape` for each topic.

## Edge Cases to Cover

- Topic with one or more non-deleted replies: `lastPostAuthor` equals the most recent reply author.
- Topic with replies where the latest is soft-deleted: `lastPostAuthor` reflects the most recent NON-deleted reply.
- Topic with all replies soft-deleted: `lastPostAuthor` is `null`.
- Topic with no replies at all: `lastPostAuthor` is `null`.
- Empty page (no topics): no crash, empty array returned.
- Nonexistent board ID: 404 with `TOPIC_NOT_FOUND_MESSAGE`.
- Gated board (e.g., visibility='members'): identical 404 oracle parity.
