## Acceptance Criteria (must test ALL of the following)

### Set-avatar: session/auth
- AC-SETAVATAR-401: No session ‚Üí `UnauthorizedException` (401); `setAvatar` on service NOT called.

### Set-avatar: input validation
- AC-SETAVATAR-400-MISSING: Missing `mediaId` in body (undefined body, empty object, null body) ‚Üí `BadRequestException` (400).
- AC-SETAVATAR-400-EMPTY: `mediaId` is an empty string or whitespace ‚Üí `BadRequestException` (400).
- AC-SETAVATAR-400-NONSTRING: `mediaId` is a number or boolean ‚Üí `BadRequestException` (400).
- AC-SETAVATAR-400-ORDER: 400 fires BEFORE `resolveSession` is called (DB/session not reached on bad body).

### Set-avatar: ownership/ownership enforcement (THE KEY SECURITY TEST)
- AC-SETAVATAR-403-NONEXISTENT: A media id that does not exist ‚Üí `ForbiddenException` (403).
- AC-SETAVATAR-403-WRONG-TYPE: A media id that exists but `resourceType !== 'avatar'` ‚Üí `ForbiddenException` (403).
- AC-SETAVATAR-403-FOREIGN: A valid avatar media id owned by ANOTHER user ‚Üí `ForbiddenException` (403).
  * **This is the key ownership test ‚Äî a user MUST NOT be able to set another user's media as their own avatar.**
- AC-SETAVATAR-ORACLE-PARITY: All three rejection cases produce the same error type and message (no existence oracle ‚Äî the caller cannot determine whether a foreign id exists).

### Set-avatar: success
- AC-SETAVATAR-SUCCESS: Caller's own avatar id (resourceType='avatar', same ownerUserId) ‚Üí 200; `setAvatar` called with correct `(callerId, mediaId)`; response is `{ avatarUrl: "/api/media/<id>" }`.
- AC-SETAVATAR-PROFILE: After a successful set-avatar, the id appears in the profile (avatar_media_id field used by `findPublicProfile`). Test via service-level unit test that `avatarMediaId` is updated.

### Remove-avatar
- AC-REMOVEAVATAR-401: No session ‚Üí `UnauthorizedException` (401); `removeAvatar` on service NOT called.
- AC-REMOVEAVATAR-SUCCESS: Active session ‚Üí 200; `removeAvatar` called with correct `callerId`; response is `{ avatarUrl: null }`.

### Service-level tests (in users.service.test.ts)
- AC-SERVICE-SETAVATAR-OWNERSHIP: `setAvatar` queries repository with `{ id: mediaId, resourceType: 'avatar', ownerUserId: callerId }` ‚Äî all three conditions in one query.
- AC-SERVICE-SETAVATAR-FOREIGN: When `mediaRepository.findOne` returns `null` (simulating nonexistent/wrong-type/foreign), `setAvatar` throws `ForbiddenException` ‚Äî not `NotFoundException` or anything else.
- AC-SERVICE-SETAVATAR-UNIFORM-MESSAGE: The ForbiddenException message is the same regardless of why the lookup failed (oracle parity).
- AC-SERVICE-SETAVATAR-SUCCESS: When `findOne` returns a matching entity, `usersRepository.update` is called with `{ avatarMediaId: mediaId }` and the method returns the `/api/media/<id>` URL.
- AC-SERVICE-REMOVEAVATAR: `removeAvatar(callerId)` calls `usersRepository.update({ id: callerId }, { avatarMediaId: null })`.

## Key Implementation Details

### Controller setAvatar handler
```
// Security order: 400 ‚Üí 401 ‚Üí 403 ‚Üí persist
1. 400 guard: body must be object with non-empty string `mediaId`
2. authService.resolveSession({ cookieHeader }) ‚Äî 401 if no session
3. usersService.setAvatar(session.user.id, mediaId) ‚Äî 403 on any not-allowed case
4. return { avatarUrl }
```

### Service setAvatar method
```typescript
async setAvatar(callerId: string, mediaId: string): Promise<string> {
  const media = await this.mediaRepository.findOne({
    where: { id: mediaId, resourceType: 'avatar', ownerUserId: callerId }
  });
  if (!media) throw new ForbiddenException("Media id not found or not usable as your avatar.");
  await this.usersRepository.update({ id: callerId }, { avatarMediaId: mediaId });
  return `/api/media/${mediaId}`;
}
```

### Service removeAvatar method
```typescript
async removeAvatar(callerId: string): Promise<void> {
  await this.usersRepository.update({ id: callerId }, { avatarMediaId: null });
}
```

### Oracle parity
`setAvatar` uses a single three-condition WHERE clause so the same null result
covers all three rejection cases. The ForbiddenException message is:
`"Media id not found or not usable as your avatar."`
