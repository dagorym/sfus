## What Changed

- apps/web/components/markdown-renderer.tsx
  - sanitizeUrl() now REJECTS any URL containing attribute-breaking characters
    (", ', <, >, &) by returning "#". Previously the function returned the raw
    URL, allowing a '"' to break out of href="..." / src="..." even after
    &quot; encoding (because the HTML parser resolves &quot; → " terminating
    the attribute).
  - convertMarkdownToHtml is now exported (@internal) to allow pure-function testing.

- apps/web/components/authoring-components.spec.ts
  - Added 9 behavioral XSS tests under describe block:
    "MarkdownRenderer XSS behavioural tests — attribute breakout (ST16 security)"
    Tests feed the proven payload and variants, asserting INERT output:
    1. Proven payload: [click](/a" onpointerover=alert`1`) → href="#", no handler
    2. Variant: quote in href rejected to #
    3. Proven payload: ![alt](/x" onerror=alert`1`) → src="#", no handler
    4. Variant: quote in img src rejected
    5. javascript: scheme in href rejected
    6. javascript: scheme in src rejected
    7. data: URI in href rejected
    8. Clean URL passes through intact (regression guard)
    9. HTTPS URL passes through intact (regression guard)
  - Updated 2 source-audit assertions that referenced old 'return "#"' literal.

- apps/web/app/forums/forums.spec.ts
  - Updated 2 source-audit assertions that referenced old 'return "#"' literal.
