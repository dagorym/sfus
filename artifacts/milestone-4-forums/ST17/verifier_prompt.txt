## ST17 Scope

ST17 implements the user-facing public profile system:

1. **`UserAvatar` component** (`apps/web/components/user-avatar.tsx`)
   - Displays a user's avatar image from the `/api/media/<id>` gated path
   - Falls back to a circular initials placeholder when no avatar or on image load error
   - `onError` → `setImgError(true)` → initials fallback (no broken image, no raw URL leak)
   - `deriveInitials()` and `resolveAvatarSrc()` exported for unit-testability
   - Styles: `apps/web/components/user-avatar.module.css`

2. **Public profile page** (`apps/web/app/users/[username]/page.tsx`)
   - URL: `/users/<username>` — publicly accessible, no authentication required
   - Fetches from `/api/users/<username>` with the five-field `profileProjection()` allowlist
   - Fields: username, displayName, bio, avatarMediaId, createdAt
   - Renders `UserAvatar`, display name, bio, and member-since date

3. **Avatar management in `/profile`** (`apps/web/app/profile/page.tsx`)
   - Uses `ImageUpload(resourceType='avatar')` for upload and remove
   - Integrated with existing profile settings page

4. **Mention-autocomplete byline** (`apps/web/components/mention-autocomplete.tsx`)
   - Updated to display `UserAvatar` in the mention suggestion list

5. **Lint remediation** (`apps/web/components/user-avatar.tsx`, line 102)
