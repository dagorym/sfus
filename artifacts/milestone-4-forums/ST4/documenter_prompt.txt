## Implementation summary

The implementer added two new public endpoints to the forums API on branch ms4-st4-implementer-20260608 (implementation commit b73d475):

### New endpoints

**POST /forums/boards/:boardId/topics** — member-authenticated topic creation
- Requires active session (401 if not present)
- Board must be publicly readable via AuthorizationService.evaluate() — nonexistent or gated boards return 404 with uniform TOPIC_NOT_FOUND_MESSAGE (oracle parity)
- Body is normalized (normalizeMarkdownBody) then validated (validateMarkdownBody) before any DB write; unsafe Markdown rejected 400
- Returns 201 with `{ topic: PublicTopicShape }`

**GET /forums/boards/:boardId/topics** — paginated topic list (no authentication required)
- Board must be publicly readable — same 404 oracle parity as above
- Returns only non-deleted topics (deletedAt IS NULL)
- Order: isPinned DESC, lastPostAt DESC, createdAt DESC (pinned topics always first)
- Pagination: page (1-indexed, default 1), pageSize (default 20, max 100)
- Returns `PaginatedTopicsShape { topics, total, page, pageSize }`

### New types

In `forums.types.ts`:
- `PublicAuthorShape` — `{ username: string; displayName: string | null }`
- `PublicTopicShape` — public-safe topic DTO (omits authorUserId, boardId, isLocked, movedByUserId, lockedByUserId, deletedAt)
- `PaginatedTopicsShape` — paginated response type
- `CreateTopicInput` — `{ boardId, title, body }`
- `TopicListQuery` — `{ page?, pageSize? }`

### Service additions in forums.service.ts

- `ForumsService.TOPIC_NOT_FOUND_MESSAGE` — static constant, identical error string for gated and nonexistent boards
- `ForumsService.createTopic(authorUserId, input)` — creates a topic with Markdown validation
- `ForumsService.listTopics(boardId, query)` — paginated listing with pinned ordering
- `private toTopicShape(topic)` — maps entity to PublicTopicShape, stripping internal fields
