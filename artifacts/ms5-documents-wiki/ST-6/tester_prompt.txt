## What changed

### BLOCKING-1 fix: 409 holder metadata now survives JsonExceptionFilter

The `JsonExceptionFilter` now extracts an optional `details` field from the HttpException response body and includes it at `error.details` in the HTTP response envelope. The two lock-conflict `ConflictException` throws in `docs.service.ts` now use `{ message, details: { lockedByUserId, lockExpiresAt } }` (previously used `lock` key which was stripped). The HTTP 409 response body is now:

```json
{
  "error": {
    "code": "CONFLICT",
    "message": "This page is currently locked by another user...",
    "statusCode": 409,
    "details": {
      "lockedByUserId": "<uuid>",
      "lockExpiresAt": "<ISO date>"
    }
  },
  "request": { ... }
}
```

### BLOCKING-2 fix: acquireLock 200 shape corrected

The controller `acquireLock` now returns `DocsLockResultShape` directly: `{ pageId, lock: DocsLockState }`. Previously it returned `{ lock: { pageId, lock: DocsLockState } }` (double-nested).

### WARNING-1 fix: softDeletePage lock check now inside transaction

`softDeletePage` is now fully transactional. The page load, `assertNotForeignLocked` call, child count, and status update all execute inside `pageRepository.manager.transaction`.
