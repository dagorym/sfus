TESTER HANDOFF: docs slug auto-derivation (POST /api/docs)

== Background ==

The implementer changed the POST /api/docs createPage route so that `slug` is now optional.
When slug is omitted or blank, the API derives a slug from the title (lowercase, non-alphanumeric
runs → single hyphen, leading/trailing hyphens stripped, fallback to "page" for no-alphanumeric
titles). If the derived path already exists, a numeric suffix is appended ("-2", "-3", …) until
the path is unique. The explicit-slug path is unchanged (still 409s on collision).

Files changed:
  apps/api/src/docs/docs.types.ts       — CreateDocPageInput.slug: string → slug?: string
  apps/api/src/docs/docs.controller.ts  — removed hard "slug must be a non-empty string" guard
  apps/api/src/docs/docs.service.ts     — createPage: added derive-and-suffix logic
